Account

case class Account(name: String, description: String, account_id: Option[Long], reference_id: Option[String], image_uri: Option[String], payment_limit: Option[Long], gaq_domains: Option[List[String]]) extends MutableWePayResource[Account, AccountResponse] with Product with Serializable

An instance of an Account.

Users on WePay can have multiple Accounts. API Applications can only manage accounts that they have created.

name

The name of the account.

description

The description of the account.

account_id

The Account ID, as determined by WePay. Your application should rarely need to set this value itself.

reference_id

A unique identifier of the account to your application. Must be unique per user per app.

image_uri

The URI to an image for the account.

payment_limit

The maximum individual payment that can be made to this account. (deprecated, I think)

gaq_domains

Domain for analytics tracking.
