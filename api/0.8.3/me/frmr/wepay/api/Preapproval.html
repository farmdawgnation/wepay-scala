Preapproval

case class Preapproval(account_id: Long, amount: Double, short_description: String, period: String, preapproval_id: Option[Long], reference_id: Option[String], app_fee: Option[Double], fee_payer: Option[String], redirect_uri: Option[String], callback_uri: Option[String], require_shipping: Option[Boolean], shipping_fee: Option[Double], charge_tax: Option[Boolean], payer_email_message: Option[String], payee_email_message: Option[String], long_description: Option[String], frequency: Option[Int], start_time: Option[DateTime], end_time: Option[DateTime], auto_recur: Option[Boolean], state: Option[String], mode: Option[String]) extends ImmutableWePayResource[Preapproval, PreapprovalResponse] with Product with Serializable

An instance of a WePay Preapproval.

Preapprovals are used to, well, preapprove payments within a certain date range and amount. Useful for kickstarter-like applications or subscription-based services.

account_id

The Account ID that will be receiving the payment from this preapproval.

short_description

The short description of what the user will be paying for.

period

How frequently the user can be charged. Can be: hourly, daily, weekly, biweekly, monthly, bimonthly, quarterly, yearly, or once.

preapproval_id

The preapproval ID assigned by WePay. Should rarely ever be set by your app.

reference_id

The reference ID of the preapproval. Should be unique per app per account.

app_fee

The fee your application will take on this transaction.

fee_payer

The person who pays transaction fee. One of "Payee" or "Payer". Defaults to "Payer".

redirect_uri

The URI that the user will be redirected to for completing the Preapproval flow.

callback_uri

The URI that IPN notifications will be directed to.

require_shipping

If true, the user will be required to enter shipping information.

shipping_fee

The fee for shipping.

charge_tax

If set to true, tax will be charged.

payer_email_message

A message from you that will be included in the confirmation email to the payer.

payee_email_message

A message from you that will be included in the confirmation email to the payee.

long_description

The long description of the transaction.

frequency

The number of times this preapproval can be used per period.

start_time

When the API can start charging using this preapproval.

end_time

The date the API can no longer charge using this preapproval.

auto_recur

Determines whether or not this is an auto-recurring Peapproval.

state

The state of the Preapproval.

mode

The mode the preapproval will run in. Can be "regular" or "iframe". Defaults to "regular".
